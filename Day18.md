# Day 18 - 使用 Transformers (9) - 總結(get it together)

#### 在前面幾章節我們講了很多，說了 tokenizer 是怎麼工作的、tokenizaion 的過程、轉換 input ID、padding、attention masks、多序列處理等等。

#### 但是 Transformers API 其實可以透過很方便的功能，自動處理這些操作，當我們直接將句子傳遞給你的分詞器時，會得到一個準備好傳遞給模型的輸入。

#### 簡單來說，這個 API 能夠自動處理文本的前處理過程，讓你可以更輕鬆地使用預訓練模型進行文本處理，而不需要手動處理分詞、轉換和填充等步驟。這使得使用 Transformer 模型變得更加方便且高效。

### 從標記 -> 模型

#### 這個範例中，我們看到如何使用 Transformers 的主要 API 來處理多個文本序列、padding、truncation 以及傳遞到模型中進行預測
```python
import torch
from transformers import BertTokenizer, BertForSequenceClassification

checkpoint = "bert-base-cased"
tokenizer = BertTokenizer.from_pretrained(checkpoint)
model = BertForSequenceClassification.from_pretrained(checkpoint)
sequences = [
    "My mood is always so good",
    "Listening to my favorite music makes me feel so happy"
]

tokens = tokenizer(sequences, padding=True, truncation=True, return_tensors="pt")
output = model(**tokens)
print(output)
```

```python
SequenceClassifierOutput(

                   [-0.7161, -0.6265]], grad_fn=<AddmmBackward0>), hidden_states=None, attentions=None
)
```
1. 我們首先載入預訓練的 BERT 模型和相對應的 tokenizer。  

2. 我們有一個包含兩個句子的序列列表 sequences。

3. 使用 tokenizer `tokenizer(sequences, padding=True, truncation=True, return_tensors="pt")`處理這些序列：
   - padding=True：表示要對序列進行填充，以確保它們具有相同的長度。
   - truncation=True：表示要對序列進行截斷，以確保它們不超過模型支持的最大長度。
   - return_tensors="pt"：表示返回 PyTorch 張量。

4. 最後，我們將處理後的輸入張量傳遞給預訓練的 BERT 模型進行預測

> 這樣的做法讓你能夠輕鬆地使用 Transformers 库來處理文本數據，而不需要手動處理分詞、填充和截斷等步驟。模型的輸入資料將會根據你的設置自動進行處理，並且你可以方便地使用預訓練模型來進行文本分類等任務。
